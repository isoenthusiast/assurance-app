## Extraction Framework Reference

### Control Template
```json
{
  "controlId": "PROCESSAREA-SUBPROCESS-###",
  "processArea": "String",
  "subProcess": "String",
  "controlName": "Clear, concise title",
  "controlStatement": "[Who] [What] [When] [Where] [Why] [How] [Evidence]",
  "controlType": "Procedural|Engineering|Analytical|Administrative|Informational|Behavioural",
  "isHSSECritical": true|false,
  "keyActivities": ["Activity 1", "Activity 2", ...],
  "riskAddressed": "Description of risk mitigated",
  "testingApproach": "How assurance would verify this control",
  "practiceDocumentName": "Exact source file name"
}
```

### Control Type Guidelines
- **Procedural**: Work steps, checklists, approval processes, communication protocols
- **Engineering**: Equipment design, protective systems, instrumentation, safety devices
- **Analytical**: Risk assessment, testing, verification, monitoring, data review
- **Administrative**: Roles, responsibilities, competency, documentation, record-keeping
- **Informational**: Training, awareness, knowledge sharing, communication content
- **Behavioural**: Team conversations, leadership engagement, culture, psychological safety

### Key Activities Best Practices
- 5-7 specific, actionable activities
- Observable in field audit
- Not exhaustive list, but critical steps
- Linked to control statement evidence requirements

### Testing Approach Best Practices
- Practical for field assurance teams
- Observable evidence (documents, records, interviews)
- Verifiable during site visits
- Proportional to HSSE criticality
